# Техническое задание

## 1. Описание задачи
Разработать CLI-утилиту, которая запрашивает текущую погоду для указанного города через публичный API (например, Open-Meteo) и выводит результат в терминал.

Утилита должна использовать библиотеку `requests` для отправки GET-запроса и обработки ответа.

## 2. Цель работы
Освоить базовое использование `requests`: отправка запроса, обработка JSON-ответа, работа с параметрами и обработка ошибок.

## 3. Функциональные требования
- Принимать название города как аргумент командной строки
- Преобразовывать город в координаты (через геокодер Open-Meteo)
- Запрашивать текущую погоду по координатам
- Выводить температуру, погодные условия и время обновления
- Корректно обрабатывать ошибки сети и HTTP

## 4. Нефункциональные требования
- Код должен быть читаемым и модульным
- Использовать только `requests` и стандартную библиотеку
- Устанавливать тайм-аут для запросов (5 секунд)
- Не использовать API-ключи (только бесплатные публичные эндпоинты)

## 5. Ограничения и допущения
- Используется бесплатный API Open-Meteo (https://open-meteo.com/)
- Город ищется по точному совпадению (без fuzzy search)
- Поддерживаются только крупные города (для которых есть данные)

## 6. Источник ТЗ
Самостоятельная постановка.
