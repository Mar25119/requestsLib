# requests

Библиотека `requests` упрощает отправку HTTP-запросов в Python. Она предоставляет интуитивный, читаемый и мощный API для взаимодействия с веб-сервисами: GET, POST, заголовки, параметры, JSON, файлы, сессии, обработка ошибок и многое другое.

Это де-факто стандарт для работы с REST API, веб-скрапинга (в простых случаях) и интеграции с внешними сервисами.

---

## Модуль 1. Контекст и предпосылки

### 1. Ограничения стандартных средств
Модуль `urllib` из стандартной библиотеки громоздкий, требует ручной обработки кодировок, заголовков и исключений. Код получается многословным и сложным для чтения.

### 2. Проблема, которую решает инструмент
`requests` делает HTTP-взаимодействие простым: «для людей, а не для машин». Он скрывает сложность, но сохраняет контроль над деталями при необходимости.

### 3. Инженерная / визуальная грамотность
Инструмент формирует культуру **чистого, выразительного кода**: запросы читаются как предложения на английском языке.

### 4. Применение в реальных проектах
Используется повсеместно:
- Telegram-боты (отправка сообщений через API)
- Интеграция с платежными системами
- Получение данных из публичных API (погода, курсы валют, новости)
- Тестирование backend-сервисов
- Автоматизация веб-взаимодействий

---

## Модуль 2. Основные идеи и механизмы

### 5. Центральные объекты и архитектура
Основные функции: `requests.get()`, `requests.post()`. Ответ — объект `Response` с атрибутами `.status_code`, `.json()`, `.text`, `.headers`.

### 6. Ключевые механизмы работы
Поддержка:
- Параметров URL (`params`)
- Заголовков (`headers`)
- JSON-тела (`json=`)
- Тайм-аутов (`timeout`)
- Обработки ошибок (`.raise_for_status()`)

### 7. Работа со структурированными данными
Автоматическая десериализация JSON через `.json()`. Поддержка кодировок (автоопределение `response.encoding`).

### 8. Интерактивность / автоматизация
Легко встраивается в циклы, CLI-утилиты, ботов, фоновые задачи. Поддерживает сессии для сохранения cookies.

### 9. Ошибки и отладка
Чёткое разделение сетевых ошибок (`requests.exceptions.ConnectionError`) и HTTP-ошибок (4xx/5xx). Можно включить логирование или использовать `.raise_for_status()`.

### 10. Организация результата
Код становится компактным, самодокументируемым и легко тестируемым.

---

## Модуль 3. Анализ и оценка

### 11. Сильные стороны
Простота, надёжность, отличная документация, огромное сообщество, поддержка всех основных HTTP-возможностей.

### 12. Ограничения и недостатки
Не поддерживает асинхронность «из коробки» (для этого есть `httpx`). Не предназначен для парсинга HTML (нужен `BeautifulSoup` или `lxml`).

### 13. Личное мнение
`requests` — must-have в арсенале любого Python-разработчика.
