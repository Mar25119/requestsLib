# Выводы по работе

## 1. Что было изучено
Я освоил базовые операции с `requests`: отправку GET-запросов с параметрами, обработку JSON, установку тайм-аутов и обработку исключений (`raise_for_status`, `RequestException`).

## 2. С какими трудностями столкнулся
Сначала не знал, что Open-Meteo требует отдельного запроса для геокодинга. Также пришлось аккуратно обрабатывать случаи, когда город не найден (пустой `results`).

## 3. Что получилось удачно
Чистый, модульный код с чётким разделением логики. Утилита корректно реагирует на ошибки сети и отсутствие города.

## 4. Где инструмент уместен
- Интеграция с REST API
- Telegram- и Discord-боты
- Сбор данных с публичных источников
- Тестирование backend-сервисов

## 5. Где инструмент неуместен
- Высоконагруженные асинхронные приложения (лучше `httpx`)
- Парсинг HTML без дополнительных библиотек
- Работа с WebSocket или streaming-данными

## 6. Итоговая оценка
Оценка: **10/10**  
`requests` — эталон простоты и надёжности. Незаменим в повседневной разработке.
